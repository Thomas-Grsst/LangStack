Qu'est-ce que GitHub ?

GitHub est une plateforme d'hébergement de code basée sur Git qui permet le développement collaboratif de projets. Elle offre des fonctionnalités de gestion de versions, de collaboration en équipe, de revue de code, d'intégration continue et bien plus encore.

Configuration Initiale

# Installation de Git
# Ubuntu/Debian
sudo apt update && sudo apt install git

# macOS
brew install git

# Windows
# Télécharger depuis git-scm.com

# Configuration globale
git config --global user.name "Votre Nom"
git config --global user.email "votre@email.com"
git config --global init.defaultBranch main

# Configuration supplémentaire
git config --global core.editor "code --wait"  # VS Code
git config --global merge.tool vscode
git config --global pull.rebase false

# Vérification de la configuration
git config --list

# Génération de clé SSH
ssh-keygen -t ed25519 -C "votre@email.com"
# Copier le contenu de ~/.ssh/id_ed25519.pub
# et l'ajouter dans GitHub Settings → SSH and GPG keys

# Test de la connexion SSH
ssh -T git@github.com
# CLONAGE DE DÉPART
# Cloner un repository existant
git clone https://github.com/utilisateur/repo.git
git clone git@github.com:utilisateur/repo.git  # Via SSH

# Cloner avec un nom spécifique
git clone https://github.com/utilisateur/repo.git mon-projet

# Cloner une branche spécifique
git clone -b develop https://github.com/utilisateur/repo.git

# INITIALISATION D'UN NOUVEAU PROJET
# Créer un nouveau repository local
mkdir mon-projet
cd mon-projet
git init

# Lier au repository GitHub
git remote add origin https://github.com/utilisateur/repo.git

# Premier commit
echo "# Mon Projet" >> README.md
git add README.md
git commit -m "Initial commit"

# Pousser vers GitHub
git push -u origin main

# WORKFLOW DE BASE
git status                    # Voir l'état
git add .                    # Ajouter tous les fichiers
git add fichier.txt          # Ajouter un fichier spécifique
git commit -m "Message"      # Committer les changements
git push                     # Pousser vers GitHub
git pull                     # Récupérer les changements

Workflow Git Essentiel

# WORKFLOW QUOTIDIEN COMPLET

# 1. METTRE À JOUR LE CODE LOCAL
git pull origin main

# 2. TRAVAILLER SUR UNE NOUVELLE FONCTIONNALITÉ
git checkout -b feature/nouvelle-fonctionnalite

# 3. FAIRE DES CHANGEMENTS
# Éditer les fichiers...
echo "Nouvelle fonctionnalité" >> feature.txt

# 4. SUIVI DES CHANGEMENTS
git status                  # Voir les fichiers modifiés
git diff                    # Voir les changements détaillés
git diff --staged           # Voir les changements staged

# 5. AJOUTER LES CHANGEMENTS
git add .                   # Ajouter tous les fichiers
# OU
git add feature.txt         # Ajouter un fichier spécifique
git add -p                  # Ajouter interactivement

# 6. COMMITTER
git commit -m "feat: ajouter nouvelle fonctionnalité

- Implémenter la fonction X
- Ajouter les tests unitaires
- Mettre à jour la documentation"

# 7. PUSHER LA BRANCHE
git push -u origin feature/nouvelle-fonctionnalite

# 8. CRÉER UNE PULL REQUEST SUR GITHUB
# Aller sur GitHub → New Pull Request

# 9. SYNCHRONISER AVEC MAIN
git checkout main
git pull origin main
git checkout feature/nouvelle-fonctionnalite
git rebase main

# 10. RÉSOUDRE LES CONFLITS SI NÉCESSAIRE
# Éditer les fichiers en conflit...
git add .
git rebase --continue

# 11. METTRE À JOUR LA BRANCHE DISTANTE
git push -f

# 12. APRÈS MERGE SUR GITHUB
git checkout main
git pull origin main
git branch -d feature/nouvelle-fonctionnalite

Gestion des Branches

# GESTION DES BRANCHES

# Lister les branches
git branch                  # Branches locales
git branch -r              # Branches distantes
git branch -a              # Toutes les branches

# Créer et switcher de branche
git branch nouvelle-branche
git checkout nouvelle-branche
git checkout -b nouvelle-branche  # En une commande

# Convention de nommage
git checkout -b feature/authentification-oauth
git checkout -b fix/correction-bug-login
git checkout -b hotfix/urgence-production
git checkout -b docs/ajout-documentation-api

# Supprimer des branches
git branch -d branche-locale      # Suppression sûre
git branch -D branche-locale      # Suppression forcée
git push origin --delete branche-distante

# Synchroniser les branches distantes
git fetch --all                   # Récupérer toutes les branches
git checkout -b local-branch origin/remote-branch

# Suivre une branche distante
git branch -u origin/ma-branche ma-branche

# RENOMMER DES BRANCHES
git branch -m ancien-nom nouveau-nom    # Locale
git push origin :ancien-nom nouveau-nom # Distante
git push origin -u nouveau-nom
# STRATÉGIES DE BRANCHING

# Git Flow
main                    # Production
develop                 # Développement
feature/feature-name    # Nouvelles fonctionnalités
release/1.2.0          # Préparation de release
hotfix/urgent-fix      Corrections urgentes

# GitHub Flow (simplifié)
main                    # Déploiement continu
feature-branches        # Toutes les fonctionnalités

# GitLab Flow
production              # Production
pre-production          # Staging
main                    # Développement
feature-branches        # Fonctionnalités

# EXEMPLES PRATIQUES

# Développer une feature
git checkout -b feature/user-dashboard
# ... travail ...
git push -u origin feature/user-dashboard

# Préparer une release
git checkout -b release/1.3.0
# ... finaliser ...
git merge release/1.3.0
git tag v1.3.0

# Correction urgente
git checkout -b hotfix/critical-security-fix
# ... correction ...
git merge hotfix/critical-security-fix

# MERGE VS REBASE
git merge feature-branch        # Préserve l'historique
git rebase main                # Historique linéaire

# Rebase interactif
git rebase -i HEAD~3          # Réorganiser les 3 derniers commits

# Stash pour changer de branche
git stash                     # Sauvegarder les changements
git checkout autre-branche
# ... travail ...
git checkout premiere-branche
git stash pop                 # Restaurer les changements

Commits et Historique

# GESTION DES COMMITS

# Commit conventionnel
git commit -m "feat: ajouter l'authentification OAuth

- Implémenter Google OAuth2
- Ajouter la configuration Firebase
- Créer les tests d'intégration

BREAKING CHANGE: modification de la structure de la base de données"

# Types de commits conventionnels
feat:     # Nouvelle fonctionnalité
fix:      # Correction de bug
docs:     # Documentation
style:    # Formatage, point-virgule manquant...
refactor: # Refactoring du code
test:     # Ajout de tests
chore:    # Tâches de maintenance

# MODIFICATION DE L'HISTORIQUE

# Modifier le dernier commit
git commit --amend -m "Nouveau message"

# Ajouter des fichiers au dernier commit
git add fichier-oublié.py
git commit --amend --no-edit

# Rebase interactif
git rebase -i HEAD~5

# Commandes disponibles:
# pick - utiliser le commit
# reword - utiliser le commit, modifier le message
# edit - utiliser le commit, mais s'arrêter pour amend
# squash - fusionner avec le commit précédent
# fixup - comme squash, mais supprimer le message
# drop - supprimer le commit

# EXEMPLE DE REBASE INTERACTIF
pick a1b2c3d feat: ajouter fonction X
pick d4e5f6g fix: corriger bug Y
pick g7h8i9j docs: mettre à jour README

# Devenir:
pick a1b2c3d feat: ajouter fonction X
fixup d4e5f6g fix: corriger bug Y
reword g7h8i9j docs: mise à jour documentation

# RÉCUPÉRATION DE COMMITS

# Voir l'historique
git log --oneline --graph --all
git log --since="2023-01-01" --until="2023-12-31"
git log --author="John"
git log -S "fonctionnalité"   # Recherche dans le code

# Réinitialiser un commit
git reset --soft HEAD~1      # Garde les changements staged
git reset --mixed HEAD~1     # Garde les changements non staged (défaut)
git reset --hard HEAD~1      # Supprime tous les changements

# Récupérer un commit supprimé
git reflog                   # Voir l'historique des actions
git checkout commit-hash     # Se positionner sur un commit
git checkout -b new-branch   # Créer une branche à partir du commit

# Stash temporaire
git stash                    # Sauvegarder les modifications
git stash list               # Lister les stashs
git stash apply              # Appliquer le dernier stash
git stash pop                # Appliquer et supprimer
git stash drop               # Supprimer le stash

Pull Requests et Code Review

# PRÉPARATION D'UNE PULL REQUEST

# 1. Synchroniser avec main
git checkout main
git pull origin main

# 2. Mettre à jour la feature branch
git checkout ma-feature
git rebase main

# 3. Résoudre les conflits si nécessaire
# Éditer les fichiers en conflit...
git add .
git rebase --continue

# 4. Pousser les changements
git push -f origin ma-feature

# Aller sur Repository → Pull Requests → New Pull Request
